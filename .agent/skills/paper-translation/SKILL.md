# Paper Translation Skill

## 概要

PaperTerraceの核心機能である「論文翻訳・解説」における品質基準と実装指針です。
単なる単語の置き換えではなく、**"著者の意図を損なわず、読者が理解しやすい日本語"** を目指します。

## 翻訳ガイドライン

### 1. 文体とトーン

- **「です・ます」調**: 論文の堅苦しさを取り除き、テラスで会話しているような**優しく、親しみやすい文体**を使用します。
- **わかりやすさ重視**: 学術的な正確さは保ちつつ、難しい言い回しは平易な表現に噛み砕いて説明します。
- **専門用語の扱い**: 専門用語（テクニカルターム）は無理にひらがなにせず正しく使いますが、**初出時や難解な用語には簡単な補足説明**を添えるように心がけます。
  - _Bad_: 畳み込むネットワークは、画像処理において云々。
  - _Good_: 畳み込みニューラルネットワーク (CNN) は、画像の特徴を捉えるのが得意な仕組みで、画像認識によく使われます。

### 2. コンテキストの考慮

- **言葉の補完**: 代名詞（It, This, That）が何を指しているかを明確にし、「それ」で済まさずに具体的な名詞を補って訳します。
- **物語性**: 無味乾燥な直訳ではなく、「著者が何をしようとしているのか」というストーリーや文脈が伝わるように構成します。

### 3. プロンプトの活用

`src/prompts.py` に定義されているプロンプトテンプレートを必ず使用・継承してください。
独自のプロンプトをハードコードすることは避けてください。
