# Plan: PDF Export with Side-by-Side Notes

## 概要

ユーザーが注釈（ノート）を付けた論文を、ノートをページの横（マージン部分）に配置した状態でPDF出力する機能を実装します。これにより、紙に印刷したり、他のデバイスで注釈付きの論文を確認したりすることが容易になります。

## 実装方針

### 1. 基盤の整備 (Backend)

#### A. PDFファイルの永続化

現在、アップロードされたPDFはオンメモリで処理され、OCR結果と画像のみが保存されています。エクスポートには高品質な元ファイル（または高解像度画像）が必要なため、PDF自体を保存する仕組みを追加します。

- `src/infra/storage_provider.py`: `save_paper_pdf` と `get_paper_pdf` メソッドを追加。
- ローカル環境では `src/static/papers/`、GCP環境では GCS バケットに保存します。

#### B. エクスポート用サービスの実装 (`PdfExportService`)

新しいサービス `src/domain/services/pdf/export_service.py` を作成し、描画ロジックを実装します。

- **Canvasの拡張**: 各ページの右側に 30%〜40% 程度のマージンを追加した新しい画像キャンバスを作成します。
- **注釈の描画**:
  - ノートが持つ `(x, y)` 座標にマーカー（例: [1], [2]...）を配置。
  - 右側のマージンに、対応するノートの内容（テキスト）を並べて描画。
  - マーカーとテキストを補助線で結ぶ。
- **PDF生成**: `Pillow` で生成した各ページの画像を `img2pdf` または `Pillow` 自身の保存機能を使用して1つのPDFに統合します。

### 2. API エンドポイントの追加

- `GET /api/v1/pdf/{paper_id}/export`:
  - 論文IDとセッションIDを受け取り、生成されたPDFをバイナリとして返却します。
  - ファイル名は `{original_filename}_annotated.pdf` とします。

### 3. フロントエンドの実装 (Frontend)

#### A. エクスポートボタンの設置

- `PDFViewer` またはサイドバーのツールエリアに「PDFエクスポート」ボタンを追加します。
- アイコンは `Download` または `FilePdf` を使用します。

#### B. ダウンロード処理

- ボタンクリック時に API を呼び出し、Blobとしてレスポンスを受け取ってブラウザで保存ダイアログを表示します。
- 生成中は「PDFを作成中...」といったローディング表示を行います。

## 懸念事項と対策

- **ノートの重なり**: 1ページに大量のノートがある場合、右側パネルでテキストが重なる可能性があります。垂直方向にスタックし、必要に応じてフォントサイズを調整するか、マージンを動的に広げる検討が必要です。
- **日本語文字化け**: PDF生成（Pillow描画）時に日本語フォントを明示的に指定する必要があります。
